Root cause analysis

The curriculum scheduler's phase transitions are catastrophically destabilizing the model. Each transition introduces new loss components without gradual blending.

⚠
Critical Issues Identified
  • Abrupt loss weight transitions — Phase 0 uses {hierarchy: 1.0}. Phase 1 instantly switches to {rotation: 1.0}. This completely restructures the optimization landscape without any blending period.
  • Learning rate remains at 1e-3 — Despite phase transitions changing the loss surface, LR stays constant. Should drop to 1e-4 and warm up again at each transition.
  • E8 depth jumping too fast — Residual depth goes 2→4→6→8 across phases. Each increase changes the model's effective capacity, causing distribution shift in latent space.
  • No checkpoints being saved — The checkpoints/kagami/ directory is empty. Zero recovery points exist. Every step is unrecoverable.

The fix

Training needs to stop and restart with these specific changes to the curriculum scheduler.

1

Stop current training

The model is actively getting worse. Every additional step is wasted compute and diverging further from any useful state.

PENDING
2

Implement gradual loss weight blending

Add 50-100 step ramp when transitioning between phases. Blend old weights → new weights linearly.

PENDING
3

Lower and reset learning rate at transitions

Base LR from 1e-3 → 1e-4. Add warmup restart at each phase transition (100 steps).

PENDING
4

Enable checkpoint saving

Save every 1,000 steps. Force save before phase transitions. Keep last 5 checkpoints.

PENDING
5

Add transition stability validation

Only advance phase when loss variance is low (stable), not just below threshold.
