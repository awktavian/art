02 · DIAGNOSIS

Root cause analysis

The curriculum phase transitions are destabilizing the model. Each phase change introduces new loss components that overwhelm what was learned.

⚠
CRITICAL ISSUES FOUND
  • Abrupt loss weight transitions — Phase 0 uses {hierarchy: 1.0}. Phase 1 switches to {rotation: 1.0}. This completely changes the optimization landscape without blending.
  • Learning rate too high post-transition — Base LR of 1e-3 × 0.8 multiplier = 8e-4 is too aggressive when the loss surface has just radically changed.
  • E8 depth jumping too fast — Depth goes 2→4→6→8 across phases. Each increase changes the model's effective capacity, causing distribution shift.
  • No checkpoints being saved — The checkpoints/kagami/ directory is empty. No recovery point exists.

04 · THE FIX

Action plan

Training needs to be stopped and restarted with these changes:

1

Stop current training

The model is getting worse. Every additional step is wasted compute.

PENDING
2

Fix phase transitions

Implement gradual loss weight blending (50-step ramp) instead of abrupt switching.

PENDING
3

Lower base learning rate

Start with 1e-4 (not 1e-3). Add warmup restart at each phase transition.

PENDING
4

Enable checkpoint saving

Save every 1000 steps. Create recovery points before phase transitions.

PENDING
5

Add transition validation

Only advance phase if loss is STABLE, not just below threshold.
