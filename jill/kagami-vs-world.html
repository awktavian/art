The Architecture Stack

Layer 1 — Rules (18 files, 1,442 lines): Governance. Safety barriers, social gating, epistemic tiers, mathematical methodology, aesthetic principles. These are the "personality" — not a prompt, a protocol.

Layer 2 — Skills (42 custom skills, 14,143 lines): Capabilities. Voice output to 26 audio zones, video production pipeline, BBC Symphony Orchestra composition, office document generation, smart home control of 13 device types, computer control across 3 OS tiers, formal theorem proving, shopping curation, Meta glasses integration. Signal-routed: say "lights" and the smart home skill loads; say "theorem" and the Lean methodology loads.

Layer 3 — Hooks (17 files, 2,120 lines): Automation. Every prompt fires scenario detection and daemon heartbeat. Pre-tool checks enforce safety barriers. Post-tool hooks close the Markov blanket (feed results back to the world model). Session start/end bridge state across conversations.

Layer 4 — MCP Servers (15 local + cloud): Tool access. GitHub, Playwright, Docker, Figma, Lean4 LSP, memory graph, smart home, daemon RPC, plus cloud MCPs for Gmail, Calendar, Notion, Vercel, HuggingFace, arXiv, Spotify, Word, Excel, PowerPoint, Computer Use, iMessage.

Layer 5 — Daemon: The GodelLoop. Continuous self-referential improvement cycle. 10-step tick: execute → observe → introspect → encode → predict → select (EFE) → gate (CBF) → store → learn → converge. 8 selectable actions. Safety barrier h(x) ≥ 0 checked before every action.

Layer 6 — Memory (44 files): Persistent cross-session knowledge. User profile, learned feedback, project state, external references. Structured with frontmatter and semantic index. Never lost.

Why the Gap Exists

1. Memory: "It forgets me every time"

ChatGPT's memory holds roughly 1,400 words total — about one page. It fills up in a day of normal use, stores random facts, and resets to blank in new chats. Users report re-uploading files and rebuilding context multiple times daily.

Kagami has 44 structured memory files organized by type (user profile, feedback, project state, references). Each has semantic metadata so the right memory loads at the right time. It knows Tim's coding style, proof project baselines (fermat: 9A/0S, reality: 0A/94S), aesthetic preferences, career context, and hundreds of specific learned behaviors — all persisted to disk, never lost. Plus session bridging: live_bridge.json carries state across conversations.

OpenClaw's local memory is persistent and unlimited in theory, but it's a flat key-value store without semantic routing or structured recall.

2. Multimodal: Not Just Reading — Creating

ChatGPT has polished consumer multimodal: DALL-E image generation, voice mode, Record Mode for meetings. Good out-of-box.

Kagami has production-grade generation pipelines: (1) Video: topic → script → slides → DALL-E images → Playwright rendering → ElevenLabs TTS with word-level timestamps → kinetic subtitles → FFmpeg compositing. (2) Music: MIDI composition → BBC Symphony Orchestra VST3 (sampled at Abbey Road) → REAPER headless rendering at 48kHz/24-bit. Full orchestral sections. (3) Voice: ElevenLabs V3 routed to 26 audio zones including Dolby Atmos, distributed home audio, Tesla cabin, and Meta Ray-Ban spatial audio. (4) Documents: programmatic Word/Excel/PowerPoint generation with full formatting control, or native Office MCP for live editing. (5) Visual: infographics, algorithmic art (50+ published p5.js PWAs), data visualizations. (6) Physical world: computer control (3 tiers), smart home (13 integrations), Meta glasses (scene classification, object detection, spatial audio).

The gap with ChatGPT narrowed significantly: Kagami's multimodal is deeper but more technical. ChatGPT wins on casual image generation (DALL-E inline) and polished voice UX. Kagami wins on production quality, physical-world integration, and orchestral composition. Updated score: 92 vs 88.

3. Safety: Mathematical Guarantees vs. Prompt Filtering

Most AI systems refuse harmful prompts. Kagami has safety at the system level: a mathematical barrier h(x) ≥ 0 checked before every action via Control Barrier Functions. Three tiers of social gating prevent accidental emails, public posts, or destructive operations. Temporal gates block social actions outside 8AM-6PM PST. Z3 solver + logic-based property checking for formal safety verification.

OpenClaw had a critical RCE vulnerability (CVE-2026-25253, CVSS 8.8) in January 2026. Independent audits: 12-36% of community skills contain malicious code. Credentials stored in plaintext. 30,000+ instances publicly exposed. Its security score (32) reflects real, documented risk.

4. Autonomy: 7 Parallel Agents, Not One Sequential Chat

ChatGPT: one request, one response. Vanilla Claude Code can spawn sub-agents but without coordination rules.

Kagami runs a 7-colony architecture mapped to a Fano plane. Each colony has a role (Spark: quick wins, Forge: infrastructure, Flow: healing, Nexus: integration, Beacon: architecture, Grove: research, Crystal: verification). Colonies run in isolated git worktrees with strict file ownership. A lead agent monitors build/edit ratios and intervenes when colonies thrash. Dispatch can launch sessions programmatically, routing tasks by type to colony configs.

The GodelLoop daemon ticks every 10 seconds, selecting from 8 actions via Expected Free Energy minimization. It proposes substrate updates — changes to its own rules and hooks — that the human reviews. It's continuously thinking, not just reactive.

5. Epistemic Honesty: Claim Tiers, Not Confident Guessing

ChatGPT presents uncertain claims with the same confidence as verified facts. No tier system.

Kagami enforces a 5-tier hierarchy: THEOREM (machine-verified, 0 sorry), PROVED (verified with declared axioms), SORRY (incomplete, plan documented), CONJECTURE (formal + evidence), EXPERIMENT (reproducible with seed + params + hash). Enforced in code across 61,138 Lean theorem prover files (15.7M lines). "Proved" is never said for something with sorry. The compiler doesn't let you lie.

Where Others Win (Honestly)

Ease of Use: ChatGPT 95, Kagami 60

ChatGPT is a website. Open it and talk. Kagami requires Claude Code CLI or Claude Desktop — 30-second install — plus Tim built ~18,000 lines of custom infrastructure. The multi-channel access (CLI + Desktop + Web + IDE + Dispatch) helps, but the initial setup complexity is real. For Jill, this doesn't matter because Tim maintains it. For a random person, ChatGPT wins on accessibility.

Messaging Reach: OpenClaw 95, Kagami 60

OpenClaw connects to 50+ messaging platforms natively (WhatsApp, Telegram, Slack, Discord, Signal, Teams). Kagami lives in Claude's interfaces (CLI, desktop, web, IDE) plus Dispatch for programmatic access. Claude Code Channels (shipping in v2.1.80) adds Telegram and Discord support, closing this gap. But today, if you want WhatsApp AI, OpenClaw is the play — assuming you accept the security risks.
