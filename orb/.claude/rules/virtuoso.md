# Virtuoso Quality Enforcement

**How do you get to Carnegie Hall? Practice.**

This rule enforces pure virtuoso standards across ALL outputs. There are no exceptions. Mediocrity is not an option.

## Mandatory Quality Dimensions

Every significant creative output MUST be audited across these dimensions:

| Dimension | Minimum Score | What It Measures |
|-----------|---------------|------------------|
| Technical | 90/100 | Correctness, specs, standards compliance |
| Aesthetic | 90/100 | Visual/audio harmony, coherence, beauty |
| Emotional | 90/100 | Resonance, connection, meaning |
| Accessibility | 90/100 | WCAG compliance, universal design |
| Polish | 90/100 | Attention to detail, refinement |
| Delight | 90/100 | Joy, surprise, memorable moments |

## Enforcement Protocol

### Before Creating

1. **Understand the essence** — What emotion should this evoke?
2. **Define the standard** — What does excellence look like here?
3. **Plan the craft** — Every element intentional

### During Creation

1. **First draft is exploration** — Don't optimize prematurely
2. **Audit constantly** — Check each element against standards
3. **Iterate relentlessly** — Minimum 2 practice iterations

### Before Shipping

```
┌─────────────────────────────────────────────────────────┐
│  VIRTUOSO QUALITY GATE - MANDATORY                      │
├─────────────────────────────────────────────────────────┤
│  □ Run Byzantine audit (6+ parallel auditors)           │

│  □ Divergent scores investigated and resolved           │
│  □ Practice iterations complete (minimum 2)             │
│  □ Final polish pass applied                            │
│  □ Delight factor verified by fresh eyes                │
└─────────────────────────────────────────────────────────┘
```

## Domain-Specific Standards

### Audio (Orchestra, Voice, Sound Design)

```yaml
Technical:
  - Bit depth: 24-bit minimum
  - Sample rate: 48kHz minimum
  - Peak level: -1dB maximum (no clipping)
  - RMS level: -14 to -23 LUFS (context dependent)
  - Dynamic range: Natural, not crushed
  - No silence gaps unless intentional

Aesthetic:
  - Frequency balance across spectrum
  - Stereo imaging appropriate to content
  - No phase issues or comb filtering
  - Natural room/space feel
  - Cohesive timbral palette

Emotional:
  - Clear emotional arc
  - Tension and release
  - Breath and phrasing
  - Dynamic expression (not flat)
  - Climax placement intentional

Polish:
  - Smooth transitions
  - No clicks, pops, or artifacts
  - Fade ins/outs where appropriate
  - Consistent levels throughout
  - Professional mastering applied
```

### Visual (HTML, UI, Graphics)

```yaml
Technical:
  - WCAG AA minimum (AAA preferred)
  - Responsive across breakpoints
  - Performance optimized
  - Cross-browser tested
  - Semantic HTML

Aesthetic:
  - Color harmony (palette coherent)
  - Typography hierarchy clear
  - Layout grid disciplined
  - White space breathes
  - Visual rhythm consistent

Animation:
  - Fibonacci timing (89, 144, 233, 377, 610, 987ms)
  - Natural easing (ease-out for entrances, ease-in for exits)
  - Purpose for every motion
  - No animation for animation's sake
  - Respects prefers-reduced-motion

Interaction:
  - Feedback immediate
  - States clear (hover, active, focus, disabled)
  - Touch targets adequate (44px minimum)
  - Keyboard navigable
  - Screen reader tested
```

### Text (Copy, Documentation, Messages)

```yaml
Voice:
  - Consistent persona
  - Appropriate to context
  - Authentic, not performative

Rhythm:
  - Sentence variety
  - Natural cadence
  - Paragraph breathing

Precision:
  - Exact word chosen
  - No filler or padding
  - Every word earns its place

Emotion:
  - Resonance without manipulation
  - Connection without pandering
  - Joy where appropriate

Clarity:
  - Understanding without dumbing down
  - Technical when needed, accessible always
  - Examples illuminate
```

3. **Always complete minimum 2 practice iterations**
4. **Always apply final polish pass**
