Button

Interactive elements for triggering actions. Buttons use glass surfaces with chromatic ring effects on hover and focus.

Live Preview

Code

<button class="prism-button">Default</button> <button class="prism-button prism-button--primary">Primary</button> <button class="prism-button prism-button--ghost">Ghost</button>

Variants

Default

Glass surface with translucent background. Use for secondary actions.

Primary

Solid gradient using the colony color (defaults to Beacon cyan). Use for primary actions.

Ghost

Transparent background with border. Use for tertiary actions or inline buttons.

Sizes

<button class="prism-button prism-button--sm">Small</button> <button class="prism-button">Medium</button> <button class="prism-button prism-button--lg">Large</button>

Icon Buttons

Use the --icon modifier for circular icon-only buttons.

<button class="prism-button prism-button--icon"> <svg>...</svg> </button> <button class="prism-button prism-button--icon prism-button--primary"> <svg>...</svg> </button>

With Icons

Buttons automatically space icons and text with an 8px gap.

Brand Colors

Use data-colony to apply different spectral colors to primary buttons.

<button class="prism-button prism-button--primary" data-colony="spark"> Spark </button>

States

Default
Hover
Active
Disabled

Props / Classes

Class Description
.prism-button Base button class (required)
.prism-button--primary Primary variant with colony gradient
.prism-button--ghost Transparent background variant
.prism-button--sm Small size (8px 16px padding)
.prism-button--lg Large size (16px 32px padding)
.prism-button--icon Circular icon-only button
data-colony="name" Sets colony color (spark, forge, flow, nexus, beacon, grove, crystal)

Accessibility

Accessibility Notes
  • Focus visible: Buttons show a chromatic ring on :focus-visible for keyboard navigation.
  • Contrast: Primary buttons maintain 4.5:1 contrast ratio for text.
  • Touch target: Minimum touch target is 44x44px for small buttons.
  • Icon buttons: Always include aria-label for icon-only buttons.
  • Disabled state: Use disabled attribute, not just visual styling.
<!-- Icon button with aria-label --> <button class="prism-button prism-button--icon" aria-label="Add item"> <svg>...</svg> </button> <!-- Disabled button --> <button class="prism-button" disabled> Cannot click </button>

Platform Variants

Desktop

Standard glass surface with backdrop blur. Chromatic ring appears on hover.

AR / VisionOS

Increased blur (40px) and lower opacity (0.4) for spatial computing. Larger touch targets recommended.

Watch

Simplified appearance with reduced blur for small screens. Focus on high contrast.

Do's and Don'ts

Do
  • Use primary buttons for the main action in a view
  • Limit to one primary button per section
  • Use colony colors consistently for semantic meaning
  • Include icons when they clarify the action
  • Provide aria-labels for icon-only buttons
Don't
  • Use multiple primary buttons in the same section
  • Mix different colony colors randomly
  • Use ghost buttons for important actions
  • Rely only on color to indicate state
  • Make buttons too small to tap on touch devices
